Technical Specification

HCP (Human-Code-AI Protocol) is an open framework for structuring context in AI-assisted development. It enables teams to treat context as production-grade code: versioned, tested, auditable, and interoperable across AI tools and platforms.

1. The .procontext/ Directory

At the core of HCP is the standardized directory structure that lives alongside your source code. This predictable layout allows AI agents to instantly locate architectural rules, ongoing sessions, and foundational knowledge without searching blindly.

  • context.md: The single source of truth. Contains critical business logic, current state, and strict invariants.
  • knowledge/: Static knowledge base. Markdown files defining APIs, domain terminology, and external integrations.
  • strategic/: Long-term vision and architecture plans.
  • sessions/: Immutable logs of AI-assisted work sessions. Each task gets its own folder for traceability.
  • decisions/: Architecture Decision Records (ADRs) generated by the AI or humans to document "why" something was built a certain way.

2. The RPI+ Workflow (States)

HCP enforces a state-machine approach to AI collaboration. Never jump straight to implementation. By explicitly declaring the state in your prompt (e.g., STATE: RESEARCH), you control the agent's behavior.

  • RESEARCH: Investigate, analyze, and understand without making code changes.
  • PLAN: Design the solution and define quality gates. Requires human approval.
  • IMPLEMENT: Write the code strictly following the approved plan.
  • VERIFY: Run tests, validate against invariants, and check metrics.
  • COMPACT: Consolidate old session logs and update context.md with new learnings.

3. Context Budget & Compaction

LLMs suffer from context window degradation. HCP solves this through Pattern 7: Thresholds. When you accumulate more than 10 sessions or the context budget exceeds 80%, the COMPACT state is triggered. The AI reads old sessions, extracts "pure gold" (patterns, architectural decisions), moves them to knowledge/ or decisions/, and archives verbose logs.

4. Roles & Vibes

Instead of relying on generic AI personas, HCP uses explicit Roles and Vibes.
Roles define the skillset (e.g., Role: ARCHITECT, Role: TESTER).
Vibes define the communication style and strictness (e.g., Vibe: PRECISO, Vibe: DIDÁCTICO).
